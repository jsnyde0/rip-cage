# ssh-bypass — composable recipe (rip-cage-wlwc.11)

The `block-ssh-bypass` PreToolUse hook is a **composable command-string guard** that
denies ssh-family commands carrying host-key-override flags:
- `-o UserKnownHostsFile=...`
- `-o GlobalKnownHostsFile=...`
- `-o StrictHostKeyChecking=accept-new|no|off`

These flags defeat the cage's known_hosts mount floor by bypassing the
`/etc/ssh/ssh_config` Match-final block. The hook blocks them at the CC PreToolUse
layer (before openssh runs) and points the agent at the legitimate path: declaring
the host in `.rip-cage.yaml ssh.allowed_hosts` or running `rc config init`.

This hook is **NOT floor** (ADR-025 D2, ADR-026 D2). It is a composable recipe.
The known_hosts/ssh_config mount floor (ADR-022) is welded in the base image and is
a distinct, separate layer — this recipe only covers the CLI-flag class.

## How to enable

1. Copy the `tools[]` entry from `manifest-fragment.yaml` into your
   `~/.config/rip-cage/tools.yaml`.
2. Run `rc build` — this bakes the hook script root-owned into the image.
3. Register the CC PreToolUse hook: merge `settings-fragment.json` into your
   `~/.claude/settings.json` or workspace `.claude/settings.json`.

The agent does the wiring by reading this recipe. No rc source edits required.

## Files

- `manifest-fragment.yaml` — TOOL recipe that bakes `block-ssh-bypass.sh` root-owned.
  Generated by `build-fragment.sh` from `hooks/block-ssh-bypass.sh`. Do not hand-edit
  the base64 blob — edit the source and re-run the generator.
- `settings-fragment.json` — CC PreToolUse hook registration fragment. Merge this into
  your CC settings to activate the hook after baking the image.
- `build-fragment.sh` — Generator: re-run to update `manifest-fragment.yaml` after
  editing `hooks/block-ssh-bypass.sh`.

## What stays in the base image

The known_hosts floor is NOT affected by this recipe:
- `COPY ssh/known_hosts.github` → `/etc/ssh/ssh_known_hosts` (baked, always present)
- `COPY ssh/ssh_config` → `/etc/ssh/ssh_config.d/00-rip-cage.conf` (baked, always present)

These provide the host-key trust list and Match-final config that the ssh-bypass hook
complements. They are unrelated to this recipe.
