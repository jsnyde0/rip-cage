# pi — composable recipe (rip-cage-wlwc.2)

The `pi-recipe` TOOL entry provisions the pi coding agent launch-hardening wrapper
and the DCG guard TypeScript extension.

## What this recipe provisions

- `/usr/local/bin/pi` — launch-hardening wrapper (precedes `/usr/bin/pi` on PATH).
  Intercepts every `pi` invocation, adds `--no-extensions -e <dcg-gate.ts>`, disabling
  auto-discovery so `/workspace/.pi/extensions/` admits NO agent-dropped extension.
- `/home/agent/.pi/agent/extensions/dcg-gate.ts` — the DCG guard extension, root-owned
  so the agent cannot overwrite or disable it (rip-cage-olen protection).
- `/home/agent/.pi/agent/extensions/` — root-owned directory (agent cannot inject
  competing extensions).
- `/etc/rip-cage/cage-pi.md` — cage-topology doc for pi, consumed by init-rip-cage.sh.

The pi binary itself (`/usr/bin/pi`) is npm-installed in the base Dockerfile and is
**not** part of this recipe. This recipe adds the wrapper + guard on top.

## How to enable

1. Copy the `tools[]` entry from `manifest-fragment.yaml` into your
   `~/.config/rip-cage/tools.yaml`.
2. Run `rc build`.

The agent does the wiring. No rc source edits required.

## Swappable guard

To run pi with ALL its extensions and NO dcg-gate, swap this recipe for
`manifest-fragment-no-guard.yaml`. Zero rc source edits needed.

## Files

- `manifest-fragment.yaml` — TOOL recipe (with DCG guard). Generated by `build-fragment.sh`.
- `manifest-fragment-no-guard.yaml` — TOOL recipe (no guard, all extensions enabled).
- `pi-wrapper.sh` — source for the launch-hardening wrapper.
- `dcg-gate.ts` — source for the DCG guard extension.
- `cage-pi.md` — source for the cage-topology doc.
- `build-fragment.sh` — generator: re-run to update `manifest-fragment.yaml` after
  editing the source files.

## How to compose (dist/default-tools.yaml model)

The published GHCR image includes this recipe via `dist/default-tools.yaml`. To compose
locally, copy the `tools[]` entry from `manifest-fragment.yaml` into your
`~/.config/rip-cage/tools.yaml` and run `rc build`. Zero rc source edits needed.
