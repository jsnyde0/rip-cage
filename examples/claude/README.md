# claude — composable recipe (rip-cage-wlwc.2)

The `claude-recipe` TOOL entry provisions the Claude Code session-isolation wrapper
and the DCG floor-lock via `/etc/claude-code/managed-settings.json`.

## What this recipe provisions

- `/usr/local/bin/claude` — session-isolation wrapper (precedes `/usr/bin/claude` on PATH).
  Resolves `CLAUDE_CONFIG_DIR` and seeds the session dir before exec-ing the real binary.
- `/etc/claude-code/managed-settings.json` — CC's managed-settings path. Hooks here
  merge un-suppressibly with user/project hooks; `PreToolUse` is deny-wins. This
  delivers the DCG guard hook via a layer the in-cage agent **cannot** edit or unregister.
  Parent `/etc/claude-code` is also root-owned (unix-dir-ownership-is-the-write-gate).
- `/etc/rip-cage/cage-claude.md` — cage-topology doc, surfaced in `~/.claude/CLAUDE.md`.

The CC binary itself (`/usr/bin/claude`) is npm-installed in the base Dockerfile and is
**not** part of this recipe. This recipe adds the wrapper + floor-lock on top.

## How to enable

1. Copy the `tools[]` entry from `manifest-fragment.yaml` into your
   `~/.config/rip-cage/tools.yaml`.
2. Run `rc build`.

The agent does the wiring by reading this recipe. No rc source edits required.

## Files

- `manifest-fragment.yaml` — TOOL recipe entry. Generated by `build-fragment.sh` from
  the canonical source artifacts. Do not hand-edit the base64 blobs — edit the source
  files and re-run the generator.
- `claude-session-wrapper.sh` — source for the session-isolation wrapper.
- `managed-settings.json` — source for the DCG floor-lock managed-settings file.
- `cage-claude.md` — source for the cage-topology doc.
- `build-fragment.sh` — generator: re-run to update `manifest-fragment.yaml` after
  editing the source files.

## How to compose (dist/default-tools.yaml model)

The published GHCR image includes this recipe via `dist/default-tools.yaml`. To compose
locally, copy the `tools[]` entry from `manifest-fragment.yaml` into your
`~/.config/rip-cage/tools.yaml` and run `rc build`. Zero rc source edits needed.
